scanf函数在%d格式下的行为

问题起源 大家都写过经典的猜数字游戏。某简单的猜数字游戏代码如下： #include <stdio.h>#include <stdlib.h> int main() { int number = rand() % 100 + 1; int guess; puts("猜数字游戏（1-100）"); for (;;) { printf("请输入你的猜测: "); scanf("%d", &guess); if (guess > number) { puts("太大了！"); } else if (guess < number) { puts("太小了！"); } else { puts("恭喜你，猜对了！"); break; } } return 0; } 编译运行效果如下： $ ./main 猜数字游戏（1-100） 请输入你的猜测: 83 太小了！ 请输入你的猜测: 85 太大了！ 请输入你的猜测: 84 恭喜你，猜对了！ $ 可以看到，输入83，84，85时，程序行为符合预期。但如果输入2147483648（超过int类型的最大值 ）,程序的输出是这样： $ ./main 猜数字游戏（1-100） 请输入你的猜测: 2147483648 太小了！ 请输入你的猜测: 这就不符合预期了，逻辑上数字2147483648应大于1~100中的任何数，程序不应该输出"太小了！"。更糟糕的是，如果我们的输入不是数字，例如输入hello，程序的行为会是这样：...
